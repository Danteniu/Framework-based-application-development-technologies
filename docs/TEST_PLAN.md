## План тестирования (кратко)

### 1) Модульные тесты (unit)
- Переходы статусов (валидные/невалидные).
- Ограничения ролей (наблюдатель не создаёт/не правит).
- Валидация вложений (лимит размера).
- Фильтрация/сортировка списка.

### 2) Интеграционные сценарии (e2e)
- Менеджер: создать дефект → назначить → перевести статусы → закрыть.
- Инженер: изменить статус у назначенного дефекта → запрет закрытия.

### 3) Нагрузочное тестирование
- Инструмент: Locust (`locustfile.py`).
- Цель: 50 активных пользователей, отклик основных страниц ≤ 1 сек (см. `README.md`).
